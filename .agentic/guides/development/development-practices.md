## A. Project layout — where code lands

- Reporter behavior → `src/index.ts` (the only entry point; `tsup.config.ts` builds exactly `entry: ['src/index.ts']`).
- Shared/public types → `src/types.ts`, re-exported from `src/index.ts`.
- Unit tests → `tests/*.test.ts` (vitest picks up `tests/**/*.test.ts` per `vitest.config.ts`).
- Build output → `dist/` is generated by tsup and gitignored (`.gitignore` lists `dist/`); never hand-edit it.
- Demo/integration project → `examples/gh-actions-demo/` (currently a stub; see its `README.md`).

Evidence: `tsup.config.ts` — single entry, `format: ['esm', 'cjs']`, `dts: true`.
Evidence: `vitest.config.ts` — `include: ['tests/**/*.test.ts']`.

## B. ESM-first with explicit `.js` extensions on relative imports

The package is `"type": "module"` and builds dual ESM/CJS. In TypeScript
source, every relative import carries the `.js` extension even though the file
on disk is `.ts`.
Evidence: `src/index.ts` — `import type { AiTriageOptions } from './types.js'`.
Evidence: `tests/reporter.test.ts` — `import AiTriageReporter from '../src/index.js'`.
Evidence: `package.json` — `"type": "module"` and the dual `exports` map (`import` → `./dist/index.js`, `require` → `./dist/index.cjs`).

## C. Public surface changes go through `src/types.ts` + re-export + changeset

The option/type surface (`AiTriageOptions`, `Classification`, `FailurePayload`,
`FailureClass`, `FailureRetry`) is defined in `src/types.ts` and re-exported
via `export type { … } from './types.js'` in `src/index.ts`. A new public
option or type is added in `src/types.ts` with a doc comment stating its
default and any truncation/redaction behavior, re-exported from `src/index.ts`,
documented in `README.md`'s Configuration table, and shipped with a changeset
(`npx changeset` — `CONTRIBUTING.md` requires one for anything user-visible).
Evidence: `src/types.ts` — every field carries a doc comment with default/limit (e.g. `/** classify at most this many failures per run; default 25 */`).
Evidence: `README.md` §Configuration — the option table mirrors `AiTriageOptions` defaults.

## D. Reporter invariant: never throw from a hook, never affect exit status

The load-bearing behavioral rule of this package: the reporter must never
throw out of a Playwright hook (`onBegin`/`onTestEnd`/`onEnd`) and must never
change the host build's exit status — with or without `failSilently`. Every
change to `src/index.ts` preserves this, and the guarding test stays green.
Evidence: `src/index.ts` — class docstring: "the reporter never throws out of a Playwright hook and never affects the build's exit status."
Evidence: `tests/reporter.test.ts` — `it('never throws from reporter hooks', …)` asserts all three hooks with empty inputs.
Evidence: `src/types.ts` — `failSilently` doc: "The reporter never fails the build either way."

## E. Unused parameters are underscore-prefixed, not deleted

Playwright's `Reporter` interface fixes hook signatures, so intentionally
unused parameters keep their position with a `_` prefix; ESLint is configured
to allow exactly that.
Evidence: `eslint.config.js` — `'@typescript-eslint/no-unused-vars': ['error', { argsIgnorePattern: '^_' }]`.
Evidence: `src/index.ts` — `onBegin(_config: FullConfig, _suite: Suite)`.

## F. Strict TypeScript, typecheck-only tsc

`tsconfig.json` is `strict: true` plus `noUncheckedIndexedAccess: true`, and
`noEmit: true` — `tsc` is the typechecker (`npm run typecheck`), tsup is the
only emitter. Do not weaken these flags; config files themselves
(`eslint.config.js`, `tsup.config.ts`, `vitest.config.ts`) are inside the
typecheck `include` and must typecheck too.
Evidence: `tsconfig.json` — `"strict": true`, `"noUncheckedIndexedAccess": true`, `"noEmit": true`, `include` covering `src`, `tests`, and the three config files.

## G. Tests: vitest, no real API calls

Unit tests run under vitest (`npm test` → `vitest run`). Tests must never call
the real Anthropic API or use real keys — `CONTRIBUTING.md` pins "tests mock
the Anthropic SDK", and the `dryRun` option exists so the examples project can
run in CI with fixture classifications at no API cost.
weak-evidence: the SDK-mocking convention is stated in `CONTRIBUTING.md` but no
mock exists yet in `tests/` (the M0 skeleton makes no API calls) — confirm the
pattern when classification code lands.
Evidence: `package.json` — `"test": "vitest run"`.
Evidence: `src/types.ts` — `dryRun` doc: "fixture classifications, no API call — for demos and CI of the examples project".

## H. Gate and CI are the same four steps

`npm run gate` = `lint && typecheck && test && build`, and
`.github/workflows/ci.yml` runs those identical four steps. Run the gate
before staging (see [`quality-gates.md`](../standards/quality-gates.md)); a
change that passes locally passes CI by construction.
Evidence: `package.json` — `"gate": "npm run lint && npm run typecheck && npm run test && npm run build"`.
Evidence: `.github/workflows/ci.yml` — steps `npm run lint` / `npm run typecheck` / `npm test` / `npm run build`.

## I. Releases go through changesets — never publish manually

Every user-visible change adds a changeset; publishing happens only via
`.github/workflows/release.yml` (changesets action on pushes to `main`, npm
provenance enabled). Agents never run `npm publish` or `npm run release`
locally — release is part of the human-gated production flow
([`git-workflow.md`](../standards/git-workflow.md) §Merge strategy,
[`escalation-policy.md`](../policy/escalation-policy.md)).
Evidence: `.github/workflows/release.yml` — `changesets/action@v1` with `publish: npm run release`.
Evidence: `.changeset/config.json` — `"access": "public"`, `"baseBranch": "main"`.

## J. Configuration model: options for behavior, env for credentials

Behavior is configured via the `AiTriageOptions` constructor object; anything
secret or environment-specific comes only from env vars (`ANTHROPIC_API_KEY`,
`GITHUB_TOKEN`, `SLACK_WEBHOOK_URL`) — never from options. New settings follow
the same split.
Evidence: `src/types.ts` — `AiTriageOptions` doc: "The full reporter option surface. Env: ANTHROPIC_API_KEY, GITHUB_TOKEN, SLACK_WEBHOOK_URL."
Evidence: `README.md` §Configuration — "Environment: `ANTHROPIC_API_KEY` (required for classification), …".
